              Encoding FIT Workout Files

              A person with knowledge of endurance training can interpret the workout instructions “Bike 60m @ HR Z2” as ride for 60 minutes in heart rate zone 2. It is also feasible to think that this person can commit this simple one step workout to memory and not need any outside guidance to perform the workout. But what if a person does not know how to interpret the workout instructions? Or what if the workout has more steps than could easily be committed to memory? For these reasons most cycling computers and wearable devices now include a workout player feature that can guide users through their workouts so that they do not need to commit them to memory.

              The FIT Workout file was designed to be able to store the most common workout instructions used for endurance training, strength training, and even yoga and pilates practices. These instructions can be used by devices to guide users through their workouts so that they do not need to commit them to memory.

              Many fitness platforms include a workout creator that lets users design workouts. These files can then be played back by devices and third party applications to guide the user through their workout. Since the platform creating the Workout file is not always aware of the platform that will playback the file, it is important that the files are well formed and robust.

              This recipe covers:

              • Overview of the FIT Workout file type.
              • Using the C# FIT SDK to create FIT Files.
              • Encoding FIT Workout Files
              • Translating common workout instructions to Workout Step messages.

              Example Project

              The example project used in this recipe is a C# console app written with .NET Core.               FIT Workout File Overview

              Workout files are used to store a list of instructions that can be used to guide a user through a structured workout. Each instruction is stored as a separate Workout Step message, and the Workout Step messages are written to the file in the order that they should bet performed. Workout files can also contain an associated sport type, workout title, and additional workout notes that can be displayed to the user.

              See the Workout File description for additional information on FIT Workout files.

              Creating FIT Files

              The steps to create a FIT file are the same regardless of the type of FIT file being created. The difference between each file type is the specific message types written to the file.

              The following steps are used to create a FIT Workout file using the FIT SDK. Steps 1–4 and 7–8 are the same for every FIT file, whereas steps 5–6 are specific to the Workout file type.

              1. Create an output stream, this can be a file or memory stream. The stream must be created with both read and write access so that the data size and CRC can be updated once all the messages have been written to the file.
              2. Create an instance of an Encode object. The Encode class is responsible writing the FIT header, calculating the data size & CRC, managing the list of active messages definitions, and writing messages to the file.
              3. Call Encode.Open() to write the FIT header to the output stream with an initial data size of zero.
              4. Write a File Id message to the output stream using Encode.Write(Mesg).
              5. Write a Workout message to the output stream using Encode.Write(Mesg).
              6. Write one or more Workout Step messages to the output stream using Encode.Write(Mesg).
              7. Once all messages are written to the file, call Encode.Close() to finalize the file with the actual data size and CRC.
              8. Close the output stream. When using a file stream the file will be saved to disk. When using a memory stream the contents of the stream should be saved before closing the stream.

              This is an example application that includes all 8 steps and creates a FIT Workout file containing the instructions for a 60 minute cycling workout that should be performed in heart rate zone 2.

              static void Main(string[] args)
{
    // 1. Create the output stream, this can be any type of stream, including a file or memory stream. Must have read/write access.
    FileStream fitDest = new FileStream(Filename, FileMode.Create, FileAccess.ReadWrite, FileShare.Read);

    // 2. Create a FIT Encode object.
    Encode encoder = new Encode(ProtocolVersion.V10);

    // 3. Write the FIT header to the output stream.
    encoder.Open(fitDest);

    // The timestamp for the workout file
    var timeCreated = new Dynastream.Fit.DateTime(System.DateTime.UtcNow);

    // 4. Every FIT file MUST contain a File ID message as the first message
    var fileIdMesg = new FileIdMesg();
    fileIdMesg.SetType(Dynastream.Fit.File.Workout);
    fileIdMesg.SetManufacturer(Manufacturer.Development);
    fileIdMesg.SetProduct(ProductId);
    fileIdMesg.SetSerialNumber(timeCreated.GetTimeStamp());
    fileIdMesg.SetTimeCreated(timeCreated);
    encoder.Write(fileIdMesg);

    // 5. Every FIT Workout file MUST contain a Workout message as the second message
    var workoutMesg = new WorkoutMesg();
    workoutMesg.SetWktName("Bike Workout");
    workoutMesg.SetSport(Sport.Cycling);
    workoutMesg.SetSubSport(SubSport.Invalid);
    workoutMesg.SetNumValidSteps(1);
    encoder.Write(workoutMesg);

    // 6. Every FIT Workout file MUST contain one or more Workout Step messages
    var workoutStepMesg = new WorkoutStepMesg();
    workoutStepMesg.SetMessageIndex(0);
    workoutStepMesg.SetWktStepName("Endurance Ride");
    workoutStepMesg.SetNotes("Keep your HR in Zone 2 for the entire ride.");
    workoutStepMesg.SetIntensity(Intensity.Active);
    workoutStepMesg.SetDurationType(WktStepDuration.Time);
    workoutStepMesg.SetDurationValue(36000000);
    workoutStepMesg.SetTargetType(WktStepTarget.HeartRate);
    workoutStepMesg.SetTargetValue(2);
    encoder.Write(workoutStepMesg);

    // 7. Update the data size in the header and calculate the CRC
    encoder.Close();

    // 8. Close the output stream
    fitDest.Close();

    Console.WriteLine($"Encoded FIT file {Filename}");
}

              Workout File Message Types

              Workout files have three required message types: File Id, Workout, and Workout Step message types. The File Id message includes information about the platform that created the file. The Workout message provides a summary of the workout and the number of workout steps contained in the file. The Workout Step message is used to define the instructions for each step of the workout.

              See the Workout File description for the list of required messages types, and a description of the fields for each.

              Sequencing Workout File Messages

              There is an expected message sequence for Workout files that should be strictly adhered to. The first message in the file should be a File Id message with the file type set to 5 (Workout). The second message in the file is a single Workout message with the num_valid_steps field set to the number of Workout Step message contained in the file. The Workout Step messages are then written to the file in the order that they should be performed. Each Workout Step message should have a zero based index that can be used as a reference for repeating steps within the workout.

              The sequence of messages for a Workout file is:

              1. File Id [1]
              2. Workout [1]
              3. Workout Step [1…n]

              Sport Types

              When creating Workout files for use on Garmin devices, only a subset of sport and sub-sport type combinations are supported. If an unsupported sport and sub-sport combination is encountered, then the device may not display the workout to the user.

              These are the supported sport type and sub-sport type combinations supported by Garmin devices.

              Sport FIT Sport Type FIT Sub Sport Type Supported Devices

To simplify this, the example project includes helper methods that can be used to create Workout Step messages. These helper methods allow for the reuse of often repeated code which makes the code easier to read and more robust.

              The CreateWorkoutStep() helper method creates a Workout Step messages and sets the field values. The default values simplify the use of this method since not every parameter needs to be provided when calling the method.

				String name = null,
				String notes = null,
				Intensity intensity = Intensity.Active,
				WktStepDuration durationType = WktStepDuration.Open,

				uint? customTargetValueHigh = null)

              Calling CreateWorkoutStep() using the default values will create a Workout Step message with an Open duration, no target values and the Active intensity type. The duration type Open is commonly implemented on devices as “continue until lap button press”.

              // Create an Active workout step with an open duration type and no target value.
var workoutStep = CreateWorkoutStep(messageIndex);

              The CreateWorkoutStepRepeat() helper method simplifies creating Workout Step messages that define repeat-blocks.

              WorkoutStepMesg CreateWorkoutStepRepeat(int messageIndex, uint repeatFrom, uint repetitions)

              The repeatFrom parameter is the message_index of the first step in the repeat block, and repetitions is the number of times to repeat the block.

              Tempo Bike

              Tempo Bike is a common cycling workout that consists of a warm up, an extended period in power zone three, and then a cool down. In this example, an open ended duration step is used to allow for a sufficient cool down.

              Using the CreateWorkoutStep() helper method, the steps for this workout can be described in three lines of code. One for each step of the workout.

              The example project takes advantage of the C# Named and Optional Arguments features. Named Arguments allows arguments to be passed to a method based on the parameter’s name rather than its position in the parameter list, and Optional Arguments allow arguments to be omitted and the default values used.

              var workoutSteps = new List<WorkoutStepMesg>();

workoutSteps.Add(CreateWorkoutStep(messageIndex: workoutSteps.Count,
                                    durationType: WktStepDuration.Time,
                                    durationValue: 600000, // milliseconds
                                    targetType: WktStepTarget.HeartRate,

                                    intensity: Intensity.Warmup));

workoutSteps.Add(CreateWorkoutStep(messageIndex: workoutSteps.Count,
                                    durationType: WktStepDuration.Time,
                                    durationValue: 2400000, // milliseconds
                                    targetType: WktStepTarget.Power,
                                    targetValue: 3));

workoutSteps.Add(CreateWorkoutStep(messageIndex: workoutSteps.Count,
                                    intensity: Intensity.Cooldown));

              The example project stores the Workout Step messages in a List collection. The Count property of the List provides the zero based message_index required for each Workout Step message. The Count property is also used as the required num_valid_steps value for the Workout message. This makes it easy to add new Workout Steps without having to manage the message_index.

              800 Meter Repeats

              800 meter repeats is a common running workout that is typically performed on a 400 meter running track. The workout consists of a warm up, a set of 800 meter efforts with a recovery in between each, and a cool down. One way to create this workout is to create individual Workout Step messages for each active and rest step of the workout. This would require 10 Workout Steps messages to describe the 5x800 meter repeats with a 200 meter recovery in between, plus two additional steps for the warm up and cool down. A better way to do this is to use a repeat step. In this example, using a repeat step reduces the number of Workout Steps from 10 to 3, not counting the warm up or cool down steps. Repeat steps also allow the device to display a count to the user so that they know how many repeats they have left. Some devices have a maximum step count that they support and using repeat steps can help to keep the step count in the file to a minimum.

              Using the CreateWorkoutStep() and CreateWorkoutStepRepeat() helper methods, the steps for this workout can be described in a five lines of code, one for each step of the workout plus the repeat step.

              workoutSteps.Add(CreateWorkoutStep(messageIndex: workoutSteps.Count,
                                    durationType: WktStepDuration.Distance,
                                    durationValue: 400000, // centimeters
                                    targetType: WktStepTarget.HeartRate,
                                    targetValue: 1,
                                    intensity: Intensity.Warmup));

workoutSteps.Add(CreateWorkoutStep(messageIndex: workoutSteps.Count,
                                    durationType: WktStepDuration.Distance,
                                    durationValue: 80000, // centimeters
                                    targetType: WktStepTarget.HeartRate,
                                    targetValue: 4));

workoutSteps.Add(CreateWorkoutStep(messageIndex: workoutSteps.Count,
                                    durationType: WktStepDuration.Distance,
                                    durationValue: 20000, // centimeters
                                    targetType: WktStepTarget.HeartRate,
                                    targetValue: 2,
                                    intensity: Intensity.Rest));

workoutSteps.Add(CreateWorkoutStepRepeat(messageIndex: workoutSteps.Count, repeatFrom: 1, repetitions: 5));

workoutSteps.Add(CreateWorkoutStep(messageIndex: workoutSteps.Count,
                                    durationType: WktStepDuration.Distance,
                                    durationValue: 100000, // centimeters
                                    targetType: WktStepTarget.HeartRate,
                                    targetValue: 2,
                                    intensity: Intensity.Cooldown));

              The Workout message contains the sport & sub sport types, the title of the workout, and the step count.

              Custom Target Values

              Sometimes it is preferable to set target values as an absolute value or as a relative percentage of a maximum or threshold value. This can be done using the Workout Step message’s custom_target_value_low and custom_target_value_high fields.

              Custom power and heart rate values can be given as absolute or relative values. Absolute values are provided as integer values representing beats per minute (BPM) for heart rate, or watts for power. Relative values are provided as an integer value ranging from 0 to 100% of the user’s maximum heart rate or 0 – 1000% functional threshold power (FTP). Since the ranges 0 to 100 (heart rate) and 0 to 1000 (power) are reserved for relative values, absolute heart rate and power values must be offset by 100 bpm or 1000 watts respectively.

              Custom speed and cadence targets are always absolute values and do not need to be offset. Speed targets should be provided as meters per second and scaled by 1000. Cadence targets should be given as revolutions per minute, steps per minute, or strokes per minute depending on the sport type with no offset or scale.

              When using custom target values, the target value should be set to zero.

              Using the CreateWorkoutStep() helper method, custom target values can be used with any step. CreateWorkoutStep() sets the default value for the target parameter to zero, so it does not need to be explicitly set.

              var workoutSteps = new List<WorkoutStepMesg>();

workoutSteps.Add(CreateWorkoutStep(messageIndex: workoutSteps.Count,
                                    durationType: WktStepDuration.Time,
                                    durationValue: 600000, // milliseconds
                                    targetType: WktStepTarget.HeartRate,
                                    customTargetValueLow: 235, // 135 + 100
                                    customTargetValueHigh: 255, // 155 + 100
                                    intensity: Intensity.Warmup));

workoutSteps.Add(CreateWorkoutStep(messageIndex: workoutSteps.Count,
                                    durationType: WktStepDuration.Time,
                                    durationValue: 2400000, // milliseconds
                                    targetType: WktStepTarget.Power,
                                    customTargetValueLow: 1175, // 175 + 1000
                                    customTargetValueHigh: 1195)); // 195 + 1000

workoutSteps.Add(CreateWorkoutStep(messageIndex: workoutSteps.Count,
                                    durationType: WktStepDuration.Time,
                                    durationValue: 600000, // milliseconds
                                    targetType: WktStepTarget.Speed,
                                    customTargetValueLow: 5556, // 5.556 meters/second * 1000
                                    customTargetValueHigh: 6944, // 6.944 meters/second * 1000
                                    intensity: Intensity.Cooldown));

              The Workout message contains the sport & sub sport types, the title of the workout, and the step count.

              Swim Workouts

              Workout files can also be used to describe lap swim workouts. Many wearable devices use built in accelerometers to detect flip and open turns while lap swimming. This allows devices to count the lengths of the pool that have been covered, track total distance of the swim activity, and calculate the average pace for each length and the overall activity. More advanced wearable devices will use the accelerometers to detect stroke type, count the number of strokes, and have a drill mode for kick sets & other swim drills. Wearable devices can use these metrics and the workout instructions in Workout files to guide users through their swim workouts.

              When creating Workout files for lap swimming, the sport type and sub-sport type combination “Swimming - Lap Swimming” should be used. The pool length in meters and pool length unit fields are also required.

              workoutMesg.SetWktName("Pool Swim");
workoutMesg.SetSport(Sport.Swimming);
workoutMesg.SetSubSport(SubSport.LapSwimming);
workoutMesg.SetPoolLength(22.86f); // 25 yards
workoutMesg.SetPoolLengthUnit(DisplayMeasure.Statute);

              For active steps, the duration type should always be set to Distance and the value should be in meters.

              var workoutStepMesg = new WorkoutStepMesg();
workoutStepMesg.SetMessageIndex(0);
workoutStepMesg.SetIntensity(Intensity.Active);
workoutStepMesg.SetDurationType(WktStepDuration.Distance);
workoutStepMesg.SetDurationDistance(365.76f); // 400 yards

              Optionally, a swim stroke can be prescribed for active steps; and any equipment that should be used for the set such as fins, kick board, paddles, buoy, or snorkel can be specified.

              workoutStepMesg.SetTargetType(WktStepTarget.SwimStroke);
workoutStepMesg.SetTargetValue(SwimStroke.Freestyle);
workoutStepMesg.SetEquipment(WorkoutEquipment.SwimPullBuoy);

              For rest steps, the duration_type field should be set to one of the following values.

              Duration Type Duration Value Usage
              Open 0 Rest until the lap button on the device is pressed.
              Time Rest Duration in Seconds Rest for a fixed amount of time in between steps
              Repetition Time Total Active + Rest Duration in Seconds When used in a rest step that is part of a repeat block a “repeat on” interval is created. The duration value represents the total active and rest duration for the repeat block, and the rest period will fluctuate based on the active duration.

              Rest steps should also use the intensity_type Rest.

              var workoutStepMesg = new WorkoutStepMesg();
workoutStepMesg.SetMessageIndex(1);
workoutStepMesg.SetIntensity(Intensity.Rest);
workoutStepMesg.SetDurationType(WktStepDuration.RepetitionTime);
workoutStepMesg.SetDurationTime(120.0f);

              The CreateWorkoutStepSwim() and CreateWorkoutStepSwimRest() helper methods included in the example project simplify creating Workout Step messages for lap swim workouts. The CreateWorkoutStepRepeat() helper method can also be used when creating swim workouts.

              A swim workout typically consists of a warm up, drills, a main set, and a cool down.

              The example project uses the CreateWorkoutStepSwim(), CreateWorkoutStepSwimRest(), and CreateWorkoutStepRepeat() to create the sample swim workout.

              var workoutSteps = new List<WorkoutStepMesg>();

// Warm Up 200 yds
workoutSteps.Add(CreateWorkoutStepSwim(messageIndex: workoutSteps.Count,
                                    distance: 182.88f,
                                    intensity: Intensity.Warmup));
// Rest until lap button pressed
workoutSteps.Add(CreateWorkoutStepSwimRest(messageIndex: workoutSteps.Count));

// Drill w/ kickboard 200 yds
workoutSteps.Add(CreateWorkoutStepSwim(messageIndex: workoutSteps.Count,
                                    distance: 182.88f,
                                    swimStroke: SwimStroke.Drill,
                                    equipment: WorkoutEquipment.SwimKickboard));
// Rest until lap button pressed
workoutSteps.Add(CreateWorkoutStepSwimRest(messageIndex: workoutSteps.Count));

// 5 x 100 yds on 2:00
workoutSteps.Add(CreateWorkoutStepSwim(messageIndex: workoutSteps.Count,
                                    distance: 91.44f,
                                    swimStroke: SwimStroke.Freestyle));

workoutSteps.Add(CreateWorkoutStepSwimRest(messageIndex: workoutSteps.Count,
                                    durationType: WktStepDuration.RepetitionTime,
                                    durationTime: 120.0f));

workoutSteps.Add(CreateWorkoutStepRepeat(messageIndex: workoutSteps.Count, repeatFrom: 4, repetitions: 5));

// Rest until lap button pressed
workoutSteps.Add(CreateWorkoutStepSwimRest(messageIndex: workoutSteps.Count));

// Cool Down Up 100 yds
workoutSteps.Add(CreateWorkoutStepSwim(messageIndex: workoutSteps.Count,
                                    distance: 91.44f,
                                    intensity: Intensity.Cooldown));

              Dynamic Fields

              The helper functions for creating Workout Steps for swim workouts make use of Dynamic fields found within the Workout Step message. Dynamic Fields provide alternate field interpretations known as subfields. Subfields can have different scale and offset values than the main field. For Workout Step messages, subfields provide access to the duration_value field using an alternative name, scale, and offset value.

              Duration Type Duration Value Units Subfield Subfield Units
              Time milliseconds duration_time seconds
              Distance centimeters duration_distance Meters

              Using subfields can make the intent of the code more clear. When looking at the following two lines of code, without the context of the duration_type, 480000 could be interpreted as milliseconds or centimeters. With the second line of code it is clear that 480.0 is a unit of time not distance.

              workoutStepMesg.SetDurationValue(480000);
workoutStepMesg.SetDurationTime(480.0f);

              See the Workout File description or Profile.xlsx for a complete listing of duration_value and target_value subfields, and the FIT File Protocol section of this guide for a description of Dynamic Fields.
